# Project Skills

Procedure skills live in `.agents/skills/`. They are vendor-neutral Markdown
instructions that agents must read before specialized work.

| Skill | Use when |
| --- | --- |
| `project-milestone` | Executing an approved milestone or plan phase |
| `project-qa` | Testing, verification, browser QA, production smoke, or QA records |
| `project-design` | Building or reviewing UI, responsive behavior, accessibility, or visual polish |
| `project-architecture` | Changing system shape, data model, APIs, integrations, or deployment |
| `project-release` | Preparing releases, tags, deploys, rollback notes, or release notes |

Manual invocation example:

```text
Use the project-qa skill to verify the signup flow on the preview deployment.
```
